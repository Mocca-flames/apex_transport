# Fix: White flash during desktop coverage-slide transitions

## Root Cause

When `setActive()` in `js/coverage.js` toggles slides:

1. **Old slide loses `.is-active`** → instantly snaps to `opacity: 0` because the `transition: opacity 0.5s` is defined *only* on `.is-active`, not on the base `.coverage-slide`. When `.is-active` is removed, the transition is removed too — no smooth fade-out.

2. **New slide gains `.is-active`** → fades in over 0.5s.

3. **Gap**: Between the old slide snapping out (opacity 0 instantly) and the new slide fully fading in, there's a ~500ms window where **no slide has any background paint**. Since `.coverage-sticky` has no `background` set, the white page background shows through = **white flash**.

## Fix

### 1. `css/components.css` — Add background to `.coverage-sticky`

Prevent page background from showing through when all slides are transparent:

```css
.coverage-sticky {
  background: var(--apex-navy);
}
```

### 2. `css/components.css` — Add transition on base `.coverage-slide`

Ensure the old slide fades out smoothly (not instantly) when it loses `.is-active`:

```css
.coverage-slide {
  transition: opacity 0.5s ease-out;
}
```

Remove `transition` from `.coverage-slide.is-active` since it's now on the base class.

### 3. `css/components.css` — Apply same fix in mobile media query

Remove `transition: opacity 0.5s ease-out` from `.coverage-slide.is-active` inside `@media (max-width: 1024px)`. The base `.coverage-slide` transition above already covers it.

## Verification

- Scroll through the coverage section on desktop — no white flash between slide transitions
- Verify mobile coverage-selector (≤1024px) is unaffected
- Verify `prefers-reduced-motion` override still works correctly
