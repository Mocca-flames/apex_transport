# Mobile Coverage Section Redesign — Interactive Country Pills

## Problem
Current scroll-jacking coverage section is broken on mobile:
- 8×100svh = 800svh track is excessive
- Address bar resize breaks scroll calculations
- Being forced through 8 slides feels tedious and overwhelming

## Goal
Replace mobile scroll-jacking with a compact interactive selector (~70vh) that lets users explore 8 countries at their own pace via taps instead of forced scroll.

## Approach: `coverage--static` Pattern
Same pattern as `services--static`: on `≤1024px`, `coverage.js` adds a `coverage--static` class, disables all scroll-jacking logic, and instead builds an interactive selector. Desktop behavior is completely untouched.

## Files to Modify

### 1. `js/coverage.js` — Add mobile static branch
In `initCoverage()`, after the existing `setActive(0, true)` init:
- Check `window.matchMedia('(max-width: 1024px)').matches`
- If mobile: add `coverage--static` class, call `buildCoverageSelector(section)`, return early
- `buildCoverageSelector(section)`: Reads existing `.coverage-slide` elements (built by content.js), extracts country name/status/description/image from each, builds a mobile selector DOM
- Add click handlers for pill taps → swap active pill styling + update detail card
- Keep the existing `MutationObserver` to handle content.js rebuilds on mobile too

### 2. `index.html` — Add mobile selector container
Inside `<section class="coverage-journey" id="coverage">`, add a new mobile-only container:
```html
<div class="coverage-selector" aria-label="Country coverage selector">
  <!-- Populated by JS -->
</div>
```
This container only shows at `≤1024px` via CSS.

### 3. `css/components.css` — Mobile selector styles
After the existing `@media (max-width: 1024px)` coverage block (~line 1848):
- `.coverage--static` — disable scroll-jacking: track height auto, sticky position static, slides visible, progress hidden
- `.coverage-selector` — show as flex column, hide on desktop
- `.coverage-selector__map` — Fixed aspect ratio image container for active country
- `.coverage-selector__pills` — Horizontal scrollable row with `overflow-x: auto`, snap points, hide scrollbar
- `.coverage-selector__pill` — Glass pill with flag emoji + country name, active state with orange glow
- `.coverage-selector__detail` — Glass card below pills showing status badge + description + clearance time

### 4. `css/components.css` — Hide desktop elements in static mode
- `.coverage--static .coverage-track` → `height: auto`
- `.coverage--static .coverage-sticky` → `position: static; opacity: 1; pointer-events: auto; height: auto; display: block`
- `.coverage--static .coverage-progress` → `display: none`
- `.coverage--static .coverage-journey__slides` → `display: none` (or hide, selector replaces it)

## Mobile Selector DOM Structure (JS-built)
```
.coverage-selector
  .coverage-selector__map
    <img> (active country's mobile image, swaps on tap)
  .coverage-selector__pills
    .coverage-selector__pill[data-index="0"].is-active
      <span>🇿🇦</span> South Africa
    .coverage-selector__pill[data-index="1"]
      <span>🇿🇼</span> Zimbabwe
    ... (8 pills)
  .coverage-selector__detail
    .coverage-selector__status Hub
    .coverage-selector__description Johannesburg dispatch HQ...
    .coverage-selector__meta 4-10h clearance
```

## Data Flow
- `content.js` calls `buildCoverageSlides()` → populates `.coverage-journey__slides` with country slides
- `MutationObserver` in `coverage.js` detects this → rebuilds mobile selector from slide content
- JS extracts from each slide: `img.src`, country `name`, `status`, `descMap[name]` text
- Country data also available from `home.coverage.countries` in JSON with flag emojis

## What Stays Unchanged
- Desktop scroll-jacking behavior (Lenis-driven)
- All existing HTML for coverage section (slides, sticky, progress rail)
- `content.js` `buildCoverageSlides()` function
- Country image assets in `/country_mobile/` and `/experiment/`
