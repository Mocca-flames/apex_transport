# Apex Transport Mobile Optimization Plan

## Objectives
- **Eliminate unwanted left spacing** on mobile/tablet viewports.
- **Optimize copywriting** for mobile readability (concise, scannable, punchy).
- Preserve all desktop design and styling — changes only affect responsive breakpoints.

---

## Current State & Issues

### Spacing Issues
1. **Hero content left padding**: `.hero-content` uses `padding: 80px 48px 120px 80px` across all viewports. The 80px left padding remains unchanged until the 640px breakpoint, causing excessive empty space on tablets (641–1024px).
2. **Tablet spacing gaps**: Some components maintain desktop-level padding/gaps on tablets, making the layout feel loose on medium screens.
3. **Horizontal real estate**: Need to ensure all content stays within the viewport with no unintended overflow.

### Copywriting Issues
Multiple text blocks are too verbose for small screens:
- Hero subheading
- Services introduction
- Individual service descriptions (6 cards)
- How It Works step descriptions (5 steps)
- Coverage section description
- Values descriptions (4 cards)
- Tech banner body copy
- Contact section tagline
- Footer tagline

---

## Proposed Changes

### CSS — Mobile/Tablet Spacing Fixes
All changes reside **inside existing media queries** to avoid touching desktop.

**`@media (max-width: 1024px)`**
- `.hero-content` → `padding: 80px 32px 120px 32px;` (reduce horizontal from 48/80 to 32px)
- `.contact-form` → `padding: 32px 24px;` (reduce from 56px)
- `.section-body` `max-width` keep 540px but fine; ensure it doesn’t cause tightness.
- `.hero-actions` `gap` reduce to `12px` for tighter button grouping (optional).
- `.services-header` gap remains 48px; leave as is.
- `.ticker-item` padding may stay.

**`@media (max-width: 640px)`** — existing rules are solid; ensure no conflicts.

## Implementation Strategy
1. **CSS Edits**:
   - Inside `@media (max-width: 1024px)` block, add/adjust:
     - `.hero-content` padding.
     - `.contact-form` padding.
     - Optionally reduce `.hero-actions` gap to `12px`.
   - Ensure no horizontal overflow by verifying container widths.

2. **HTML Edits**:
   - Replace all identified copy blocks with the proposed shorter versions.
   - Maintain semantic structure; only change inner text.

3. **Validation Points**:
   - Check that no element overflows the viewport horizontally on mobile/tablet.
   - Confirm touch targets remain adequate (buttons >= 44px).
   - Ensure line lengths stay within 60–75 characters for comfortable reading.

---

## Expected Results
- Clean, spacious mobile layout without awkward left margins.
- Faster content consumption on small screens.
- Higher engagement and conversion potential from clearer messaging.
- Desktop remains visually and functionally identical to original.
